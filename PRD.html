Summary

미션 요약

반복적인 멀티채널 콘텐츠 변환 작업을 자동화하여
팀이 기획, 검수, 캠페인 개선에 더 많은 시간을 쓰도록 한다.

입력

Notion 단일 DB에 주제를 필수로 입력하고 키워드는 선택 입력한다.

생성

Make AI Toolkit으로 플랫폼별 텍스트를 만들고 Cloudflare Workers AI HTTP 엔드포인트로 대표 이미지를 생성한다.

저장

생성 결과는 자동 게시하지 않고 Notion 페이지 본문과 첨부 파일로 저장한다.

노코드 자동화 Notion 초안 저장 A/B 톤 생성 공통 대표 이미지 1장 자동 게시 제외 자동 품질 검수 제외

Problem

문제 정의

같은 주제를 플랫폼마다 다시 쓰는 과정은 단순 반복이지만,
글자 수와 톤앤매너, 이미지 활용 방식이 달라 자동화 설계가 필요하다.

현재 문제

  • Instagram, 블로그, Threads에 맞춰 같은 메시지를 반복 작성한다.
  • 플랫폼별 분량, 해시태그, 문체 기준이 달라 작업 시간이 늘어난다.
  • 결과물이 흩어져 팀원이 비교, 수정, 제출 상태를 파악하기 어렵다.
  • 이미지 생성 결과와 텍스트 결과가 분리되어 최종 산출물 정리가 번거롭다.

해결 방향

  • Notion DB를 콘텐츠 요청과 결과 저장의 단일 작업 공간으로 사용한다.
  • Make가 상태값을 감지해 텍스트 생성, 이미지 생성, 파일 변환, Notion 저장을 순차 실행한다.
  • A/B 톤 결과를 한 페이지에서 비교해 팀이 수동으로 최종 판단한다.
  • 자동 게시를 제외해 저작권, 브랜드, 플랫폼 정책 리스크를 낮춘다.

Goals

목표

팀 프로젝트에서 실제 구현 가능한 자동화 범위를 명확히 하고,
최종 제출물의 구조와 성공 조건을 정의한다.

워크플로우 설명 가능

하나의 주제가 플랫폼별 콘텐츠로 변환되고 Notion에 저장되는 흐름을 팀원이 설명할 수 있다.

텍스트+이미지 자동화

GPT 기반 텍스트 생성과 이미지 생성 AI 호출을 Make에서 연결한다.

플랫폼별 포맷 반영

분량, 태그, 문체, 제목 구조 등 플랫폼별 콘텐츠 요구사항을 프롬프트 설계에 반영한다.

결과 중앙화

생성 결과, 오류, A/B 평가를 Notion 단일 DB와 페이지 본문에서 확인한다.

Workflow

자동화 워크플로우

상태값 변경 자체가 외부 게시를 수행하지 않는다.
사용자가 Status를 생성요청으로 변경하면 Make 실행의 트리거가 되고, 결과는 Notion 초안으로 저장된다.

Notion 입력

사용자가 단일 DB에 주제와 선택 키워드를 입력하고 Status를 생성요청으로 직접 변경한다.

Make 감지

Make가 생성요청 상태의 행을 감지하고 Status를 생성중으로 변경한 뒤 시나리오를 시작한다.

텍스트 생성

Make AI Toolkit이 Instagram, 네이버 블로그, Threads 결과를 친근함과 전문적 톤으로 생성한다.

이미지 생성

Cloudflare Workers AI HTTP 엔드포인트가 1:1 대표 이미지를 Base64 응답으로 반환한다.

파일 변환

Make가 Base64 값을 이미지 파일로 변환하고 Notion 페이지 첨부에 사용할 수 있는 형태로 준비한다.

본문 저장

플랫폼별 A/B 결과와 이미지 첨부를 같은 Notion 페이지 본문에 섹션으로 저장한다.

상태 업데이트

모든 단계가 성공하면 완료, 이미지 단계만 실패하면 부분완료와 오류 메시지를 기록한다.

팀 수동 확인

담당자가 Notion 페이지와 Make 실행 화면을 직접 확인하고 A/B 평가와 결론을 작성한다.

대기 생성요청 생성중 완료 부분완료 실패

부분완료 기준

텍스트 생성과 Notion 본문 저장은 성공했지만 이미지 생성, Base64 변환, Notion 첨부 중 하나가 실패한 경우 텍스트 결과를 유지하고 Status를 부분완료로 기록한다.

Content Rules

플랫폼별 생성 규칙

프롬프트 전문은 PRD에 고정하지 않는다.
대신 구현자가 지켜야 할 출력 요구사항과 수정 기록 양식을 정의한다.

대표 이미지 요구사항

  • 콘텐츠 주제를 시각적으로 설명하는 공통 대표 이미지 1장 생성.
  • 기본 비율은 1:1, 기준 크기는 1080px.
  • 실제 게시 전 사람이 저작권, 브랜드 적합성, 민감 요소를 검토한다.
  • 결과는 Base64 응답을 Make에서 파일로 변환한 뒤 Notion에 첨부한다.

프롬프트 기록 요구사항

  • 플랫폼별 프롬프트 설계 의도와 출력 조건을 기록한다.
  • 초안, 수정안, 최종안의 변경 사유를 남긴다.
  • 프롬프트 전문은 별도 문서 또는 Notion 페이지에 관리할 수 있다.
  • PRD에는 전체 프롬프트 문안이 아니라 설계 기준만 포함한다.

Bonus

A/B 테스트

보너스 과제 범위에 포함하되 자동 성과 예측은 하지 않는다.
두 톤의 결과를 생성하고 팀이 수동으로 비교한다.

A안: 친근함

말을 건네는 듯한 문장, 쉬운 표현, 공감형 메시지를 사용한다. Instagram과 Threads에서 참여 유도에 적합하다.

대화형 공감 가벼운 CTA

B안: 전문적

근거와 구조가 분명한 문장, 정보형 제목, 명확한 요약을 사용한다. 네이버 블로그와 B2B 문맥에 적합하다.

정보형 신뢰감 구조화

수동 평가 기준

팀원은 Notion 페이지에서 두 톤의 결과를 읽고 플랫폼별 적합성과 실제 활용 가능성을 평가한다.

채널 적합성플랫폼 문법과 사용 맥락에 맞는가
명확성핵심 메시지가 빠르게 이해되는가
CTA사용자 행동 유도가 자연스러운가

Acceptance

인수조건

아래 조건은 자동화된 로컬 테스트가 아니라
Notion, Make, Cloudflare Workers AI가 연결된 뒤 담당자가 수동으로 확인하는 기준이다.

ID 조건 수동 확인 방법
AC-01Notion에서 주제만 입력한 뒤 사용자가 Status = 생성요청으로 직접 변경하면 Make 실행이 시작되어야 한다.담당자가 Notion 행의 상태값과 Make 실행 화면을 직접 확인한다.
AC-02주제 + 키워드 입력 후 실행하면 Instagram, 네이버 블로그, Threads 결과가 A/B 톤별로 같은 Notion 페이지 본문에 저장되어야 한다.담당자가 Notion 페이지 본문에서 플랫폼별 섹션과 A/B 섹션을 직접 확인한다.
AC-03Cloudflare Workers AI의 Base64 이미지 응답은 Make에서 파일로 변환되어 Notion 페이지 첨부로 표시되어야 한다.담당자가 Make의 파일 변환 단계와 Notion 첨부 영역을 직접 확인한다.
AC-04이미지 단계가 실패해도 생성된 텍스트 결과는 유지되고, 상태는 부분완료, 오류 메시지는 이미지 단계 실패 사유로 기록되어야 한다.담당자가 실패 케이스 실행 후 Notion의 텍스트 결과, 상태값, 오류 메시지를 직접 확인한다.
AC-05인수조건 검증은 별도 증거 파일이나 캡처 저장 없이 사람이 매뉴얼하게 확인한다.담당자가 Notion 페이지와 Make 실행 화면을 보고 통과 여부를 판단한다.

Constraints

제약·리스크

본 프로젝트는 자동화 경험과 산출물 정리에 초점을 맞추며,
게시와 자동 품질 판단은 의도적으로 제외한다.

금지 사항

  • 저작권 문제가 있는 콘텐츠를 실제 소셜 미디어에 무단 게시하지 않는다.
  • 타인의 콘텐츠를 복사해 제출하지 않는다.
  • AI 생성 결과를 사람 검토 없이 브랜드 공식 콘텐츠로 사용하지 않는다.

제외 범위

  • Instagram, 네이버 블로그, Threads API 직접 게시.
  • AI 기반 자동 품질 검수와 자동 우수안 선정.
  • 머신 파싱용 JSON 스펙 임베드.
  • 세부 일정표와 사용자/팀 역할 섹션.
참고 기준

Instagram 해시태그 5개와 Threads 태그 1개 기준은 최근 플랫폼 변화 참고값으로 반영한다. 관련 참고 링크: Instagram hashtag limit, Threads text format reference.

본 PRD는 팀 실행과 코딩 에이전트의 요구사항 분석을 동시에 고려한 구현 기준 문서다. 외부 서비스 연결 정보, 실제 API 키, 계정 권한은 포함하지 않는다.
