Summary

미션 요약

반복적인 멀티채널 컨텐츠 변환 작업을 자동화하여
팀이 기획, 검수, 캠페인 개선에 더 많은 시간을 쓰도록 한다.

입력

공개 Notion Form DB에서 주제/키워드와 메세지 톤을 입력한다.

생성

Make AI Toolkit으로 컨텐츠 브리프를 먼저 만들고, 선택한 톤 기준으로 플랫폼별 텍스트와 대표 이미지를 생성한다.

저장

컨텐츠 브리프와 플랫폼별 컨텐츠는 응답 페이지 본문에, 이미지는 파일 속성에 저장한다.

노코드 자동화 Notion Form DB 선택 톤 생성 공통 대표 이미지 1장 자동 게시 제외 자동 품질 검수 제외

Problem

문제 정의

같은 주제를 플랫폼마다 다시 쓰는 과정은 단순 반복이지만,
글자 수와 톤앤매너, 이미지 활용 방식이 달라 자동화 설계가 필요하다.

현재 문제

  • Instagram, 블로그, Threads에 맞춰 같은 메시지를 반복 작성한다.
  • 플랫폼별 분량, 해시태그, 문체 기준이 달라 작업 시간이 늘어난다.
  • 결과물이 흩어져 팀원이 비교, 수정, 제출 상태를 파악하기 어렵다.
  • 이미지 생성 결과와 텍스트 결과가 분리되어 최종 산출물 정리가 번거롭다.

해결 방향

  • Notion Form DB를 컨텐츠 요청과 결과 저장의 단일 작업 공간으로 사용한다.
  • Make가 Responses 뷰의 신규 row를 감지해 컨텐츠 브리프, 플랫폼별 컨텐츠, 이미지 생성을 순차 실행한다.
  • 동일 주제/키워드를 톤별로 따로 제출한 결과를 팀이 수동 비교해 결론을 정리한다.
  • 자동 게시를 제외해 저작권, 브랜드, 플랫폼 정책 리스크를 낮춘다.

Goals

목표

팀 프로젝트에서 실제 구현 가능한 자동화 범위를 명확히 하고,
최종 제출물의 구조와 성공 조건을 정의한다.

워크플로우 설명 가능

하나의 주제/키워드가 컨텐츠 브리프와 플랫폼별 컨텐츠로 변환되고 Notion에 저장되는 흐름을 팀원이 설명할 수 있다.

텍스트+이미지 자동화

GPT 기반 텍스트 생성과 이미지 생성 AI 호출을 Make에서 연결한다.

플랫폼별 포맷 반영

분량, 태그, 문체, 제목 구조 등 플랫폼별 컨텐츠 요구사항을 프롬프트 설계에 반영한다.

결과 중앙화

컨텐츠 브리프, 플랫폼별 컨텐츠, 이미지 결과를 같은 Notion 응답 페이지에서 확인한다.

Workflow

자동화 워크플로우

폼 제출은 외부 게시를 수행하지 않는다.
Responses 뷰에 row가 추가되면 Make가 자동 감지하고 결과는 같은 응답 페이지에 저장된다.

공개 폼 제출

사용자가 주제/키워드와 메세지 톤을 입력해 공개 Notion 폼을 제출한다.

Responses row 감지

Form DB의 Responses 뷰에 응답 row가 추가되면 Make가 자동으로 감지해 시나리오를 시작한다.

컨텐츠 브리프 생성

Make AI Toolkit이 주제/키워드와 선택 톤을 기반으로 플랫폼별 생성에 참고할 컨텐츠 브리프를 만든다.

플랫폼별 컨텐츠 생성

컨텐츠 브리프를 레퍼런스로 인스타그램, 네이버 블로그, 스레드 컨텐츠를 선택 톤 기준으로 생성한다.

텍스트 결과 저장

컨텐츠 브리프와 플랫폼별 컨텐츠를 응답 페이지 본문에 heading과 JSON 코드 블록으로 작성한다.

이미지 생성/첨부 시도

Cloudflare Workers AI로 1:1 이미지를 생성하고, 성공 시 Base64 값을 파일로 변환해 이미지 속성에 첨부한다.

이미지 실패 처리

이미지 생성이 실패하면 Make가 3회 재시도하고, 최종 실패 시 이미지 속성은 빈 값으로 둔 채 완료로 간주한다.

팀 수동 확인

담당자가 Notion 결과와 Make 콘솔을 확인하고 필요시 별도 비교 분석 페이지에 결론을 정리한다.

Interface

Notion DB 구조

공개 폼 응답 row를 요청과 산출물의 기준 단위로 사용한다.
텍스트 결과는 응답 페이지 본문에 작성하고 이미지는 파일 속성에 첨부한다.

속성명 유형 필수 여부 설명
주제/키워드Title폼 필수컨텐츠 생성의 중심 입력. 주제와 키워드를 별개 필드로 분리하지 않는다.
메세지 톤Select폼 필수친근함 또는 전문적 중 하나를 선택한다.
이미지Files자동 입력이미지 생성 성공 시 파일을 첨부한다. 3회 재시도 후 실패하면 빈 값을 허용한다.

본문 결과 섹션 기준

  1. ## 컨텐츠 브리프 섹션에 브리프 JSON 코드 블록 작성
  2. ## 인스타그램 섹션에 Instagram 결과 JSON 코드 블록 작성
  3. ## 네이버 블로그 섹션에 네이버 블로그 결과 JSON 코드 블록 작성
  4. ## 스레드 섹션에 Threads 결과 JSON 코드 블록 작성

이미지 실패 기준

이미지 생성만 실패한 경우 Make에서 3회 재시도한다. 최종 실패해도 본문에 작성된 텍스트 결과는 유지하며, 이미지 속성은 빈 값으로 둘 수 있다.

Content Rules

플랫폼별 생성 규칙

프롬프트 전문은 PRD에 고정하지 않는다.
대신 구현자가 지켜야 할 출력 요구사항과 수정 기록 양식을 정의한다.

Instagram

형식짧은 캡션 + 해시태그
분량해시태그 포함 전체 150~500자
태그해시태그 정확히 5개
톤폼에서 선택한 메세지 톤 적용

네이버 블로그

형식제목 + 소제목 + 본문
분량본문 1,500~2,500자 내외(정보가 적으면 더 짧게 가능)
주제/키워드본문 맥락과 검색형 문맥에 자연스럽게 반영
톤선택 톤을 유지하되 읽기 쉬운 문장 사용

Threads

형식단일 게시글
분량500자 이내
태그해시태그 0~3개
톤선택 톤을 유지하되 핵심 메시지 우선

대표 이미지 요구사항

  • 컨텐츠 브리프와 주제/키워드를 시각적으로 설명하는 공통 대표 이미지 1장 생성.
  • 기본 비율은 1:1, 기준 크기는 1080px.
  • 실제 게시 전 사람이 저작권, 브랜드 적합성, 민감 요소를 검토한다.
  • 결과는 Base64 응답을 Make에서 파일로 변환한 뒤 이미지 속성에 첨부한다.
  • 이미지 생성 실패 시 Make에서 3회 재시도하며, 최종 실패 시 이미지 속성은 빈 값으로 둘 수 있다.

프롬프트 기록 요구사항

  • 플랫폼별 프롬프트 설계 의도와 출력 조건을 기록한다.
  • 초안, 수정안, 최종안의 변경 사유를 남긴다.
  • 프롬프트 전문은 별도 문서 또는 Notion 페이지에 관리할 수 있다.
  • PRD에는 전체 프롬프트 문안이 아니라 설계 기준만 포함한다.

Manual Analysis

수동 비교 분석

친근함과 전문적 결과를 한 번에 자동 생성하지 않는다.
동일 주제/키워드를 톤만 다르게 각각 제출하고 사람이 결과를 비교한다.

친근함 제출

말을 건네는 듯한 문장, 쉬운 표현, 공감형 메시지를 사용한다. Instagram과 Threads에서 참여 유도에 적합하다.

대화형 공감 가벼운 CTA

전문적 제출

근거와 구조가 분명한 문장, 정보형 제목, 명확한 요약을 사용한다. 네이버 블로그와 B2B 문맥에 적합하다.

정보형 신뢰감 구조화

분석 방식

동일한 주제/키워드를 메세지 톤만 다르게 각각 폼 제출한 뒤, 사용자는 결과물을 수동 비교해 플랫폼, 주제 등 변수별로 효과적인 컨텐츠 결론을 별도 Notion 페이지에 정리한다.

채널 적합성플랫폼 문법과 사용 맥락에 맞는가
주제 적합성주제/키워드 의도가 잘 살아나는가
톤 효과친근함 또는 전문적 톤이 목적에 맞는가
활용 결론어떤 조건에서 어떤 컨텐츠가 효과적인가

Acceptance

인수조건

아래 조건은 자동화된 로컬 테스트가 아니라
Notion, Make, Cloudflare Workers AI가 연결된 뒤 담당자가 수동으로 확인하는 기준이다.

ID 조건 수동 확인 방법
AC-01공개 Notion 폼에서 주제/키워드와 메세지 톤을 제출하면 Responses 뷰에 row가 생성되고 Make 실행이 시작되어야 한다.담당자가 Responses 뷰의 신규 row와 Make 실행 화면을 직접 확인한다.
AC-02Make는 플랫폼별 컨텐츠 생성 전에 컨텐츠 브리프를 생성해 응답 페이지 본문 ## 컨텐츠 브리프 섹션의 JSON 코드 블록으로 작성해야 한다.담당자가 Form 응답 페이지 본문의 컨텐츠 브리프 섹션을 직접 확인한다.
AC-03## 인스타그램, ## 네이버 블로그, ## 스레드 섹션에는 폼에서 선택한 메세지 톤 기준의 컨텐츠 JSON 코드 블록이 작성되어야 한다.담당자가 각 플랫폼 본문 섹션의 톤과 출력 형식을 직접 확인한다.
AC-04이미지 생성이 성공하면 이미지 속성에 파일이 첨부되고, 3회 재시도 후에도 실패하면 이미지 속성은 빈 값이어도 완료로 간주한다.담당자가 이미지 속성과 Make 콘솔 실행 이력을 직접 확인한다.
AC-05동일한 주제/키워드를 메세지 톤만 다르게 각각 제출한 결과를 수동 비교하고, 결론은 별도 Notion 페이지에 정리할 수 있어야 한다.담당자가 톤별 응답 row와 별도 비교 분석 페이지를 직접 확인한다.
AC-06인수조건 검증은 별도 증거 파일이나 캡처 저장 없이 사람이 매뉴얼하게 확인한다.담당자가 Notion 페이지와 Make 실행 화면을 보고 통과 여부를 판단한다.

Constraints

제약·리스크

본 프로젝트는 자동화 경험과 산출물 정리에 초점을 맞추며,
게시와 자동 품질 판단은 의도적으로 제외한다.

금지 사항

  • 저작권 문제가 있는 컨텐츠를 실제 소셜 미디어에 무단 게시하지 않는다.
  • 타인의 컨텐츠를 복사해 제출하지 않는다.
  • AI 생성 결과를 사람 검토 없이 브랜드 공식 컨텐츠로 사용하지 않는다.

제외 범위

  • Instagram, 네이버 블로그, Threads API 직접 게시.
  • AI 기반 자동 품질 검수와 자동 우수안 선정.
  • 세부 일정표와 사용자/팀 역할 섹션.
참고 기준

Instagram 해시태그 5개와 Threads 해시태그 0~3개 기준은 최근 플랫폼 변화 참고값으로 반영한다. 관련 참고 링크: Instagram hashtag limit, Threads text format reference.

본 PRD는 팀 실행과 코딩 에이전트의 요구사항 분석을 동시에 고려한 구현 기준 문서다. 외부 서비스 연결 정보, 실제 API 키, 계정 권한은 포함하지 않는다.
